# AI 활용 기록 (MVP 문서/개발 보조)

이 프로젝트는 MVP 개발 과정에서 사람이 작성한 산출물과 AI(Codex)를 활용해 생성/정리한 산출물이 함께 존재합니다. 이 문서는 “AI를 어떤 방식으로 사용했고, 결과물을 어떻게 검증했는지”를 개발 기록 관점에서 간단히 정리합니다.

## 사용 목적

- 문서 산출물 인벤토리화 및 분류(기획/아키텍처/API/개발노트/리포트/에셋)
- 문서 간 중복/불일치(파일 위치, 경로 참조, 명칭) 발견 및 정리
- API 연동 방식/환경변수/CORS 등 운영 설정 문서화 보조
- 작업 지시서/리뷰 문서 초안 생성 및 포맷 통일

## 사용 방식(원칙)

- AI는 “초안 생성 + 구조화 + 체크리스트 제안” 역할로 제한
- 실제 구현/정책 결정은 사람이 최종 승인
- 경로/파일 이동 등 리포지토리 구조 변경은 실행 전후로 링크/참조를 검색해 검증

## 산출물 예시

- `docs/INDEX.md`: 문서 목차(타입/타임라인 기준)
- `docs/reports/phase-2/PHASE_2_WORK_ORDER.md`: 작업 지시서(백엔드/프론트엔드 범위 명시)
- `docs/api/mvp/`: API 스펙 및 환경설정 가이드

## 한계/주의

- AI가 작성한 내용은 “현재 구현 범위”와 불일치할 수 있어, 코드/설정 파일로 근거를 확인해야 합니다.
- 운영/보안 관련 설정(CORS, 인증, 배포)은 항상 사람이 검토 후 적용합니다.
