# 2. 화면 정의서

**Cherry 모바일 웹뷰 화면 정의서 (MVP v1.1)**

## 1. 문서 목적
본 문서는 Cherry MVP를 **모바일 웹뷰 기준**으로 구현하기 위한 화면 정의서이다.

*   현재 구현된 모바일 화면을 **기준 시안**으로 확정한다
*   MVP 범위에서 **반드시 필요한 화면만 정의**한다
*   이후 프론트엔드/백엔드 RFP의 기준 문서로 사용한다

본 문서는 **데스크톱 UI, 반응형 대응, 디자인 개선을 포함하지 않는다.**

## 2. 화면 설계 기준

### 2.1 플랫폼 기준
*   모바일 웹뷰(WebView) 기준
*   앱/웹 구분 없이 “모바일 화면”만 고려

### 2.2 설계 원칙
*   MVP 핵심 흐름(탐색 → 상세 → 채팅 → 거래 완료)을 방해하지 않는다
*   화면 수는 최소화한다
*   하나의 화면은 하나의 목적만 가진다

## 4. 화면별 정의

### 4.1 글로벌 헤더 (공통)
**목적**
*   서비스 진입점 및 탐색 시작점 제공

**구성 요소**
*   서비스 로고 (홈 이동)
*   검색 입력
*   알림(후순위)
*   사용자 아이콘(마이페이지 진입)

**정책**
*   로그인 여부에 따라 아이콘 동작만 분기
*   장바구니 아이콘은 포함하지 않음

### 4.2 홈 화면
**목적**
*   “지금 이 플랫폼에서 어떤 굿즈가 거래되고 있는지”를 직관적으로 보여줌

**주요 역할**
*   첫 진입 시 상품 노출
*   탐색으로 자연스럽게 유도

**구성**
*   이벤트/배너 영역 (선택)
*   추천 상품 리스트
*   최신 상품 리스트

**비고**
*   홈은 거래를 시작하게 만드는 ‘입구’ 역할만 수행

### 4.3 상품 목록 화면 (검색/카테고리)
**목적**
*   사용자가 원하는 아이돌 굿즈를 빠르게 좁혀볼 수 있도록 함

**주요 기능**
*   검색 결과 표시
*   카테고리 기반 탐색

**구성**
*   상품 카드 리스트
*   상품 상태 뱃지(판매중/예약/완료)
*   찜 상태 표시

### 4.4 상품 상세 화면 (핵심)
**목적**
*   거래 여부를 결정하는 가장 중요한 화면

**주요 역할**
*   상품 정보 전달
*   판매자 신뢰 판단
*   채팅 시작

**구성**
*   상품 이미지 (다중)
*   상품 제목, 가격, 상태
*   판매자 정보 요약
*   태그
*   CTA 버튼
    *   찜
    *   채팅하기

**정책**
*   판매자는 상태 변경 버튼 노출
*   구매자는 채팅 CTA만 노출

### 4.5 상품 등록 화면
**목적**
*   판매자가 굿즈를 쉽게 등록할 수 있도록 함

**구성**
*   이미지 업로드
*   상품 정보 입력
*   등록 완료 CTA

**정책**
*   최소 입력만 요구
*   등록 후 상품 상세로 이동

### 4.6 채팅 목록 화면
**목적**
*   진행 중인 거래 커뮤니케이션을 한눈에 보여줌

**구성**
*   채팅방 리스트
*   상대방 정보
*   상품 요약
*   거래 상태 표시

### 4.7 채팅 상세 화면
**목적**
*   판매자–구매자 간 거래 협의

**구성**
*   메시지 리스트
*   메시지 입력 영역
*   상품 요약 영역
*   거래 상태 표시

**정책**
*   텍스트 메시지 전용
*   이미지 전송 제외(MVP)

### 4.8 로그인 / 닉네임 설정 화면
**목적**
*   사용자 식별

**구성**
*   로그인 수단
*   최초 로그인 시 닉네임 설정

**정책**
*   닉네임은 서비스 내 식별 기준

### 4.9 마이페이지
**목적**
*   사용자 개인 활동 관리

**구성**
*   내 판매글
*   내 채팅
*   찜한 상품
*   프로필 수정

### 4.10 찜 목록 화면
**목적**
*   사용자의 관심 상품 관리
*   재방문 유도

**정책**
*   커뮤니티 기능의 간접 신호로 활용

## 5. MVP에서 제외된 화면
다음 화면은 MVP v1.1에서 포함하지 않는다.

*   장바구니
*   결제/주문 화면
*   커뮤니티 게시판
*   관리자 전용 UI
*   알림 센터

## 6. 이 문서의 역할
본 화면 정의서는 다음 작업의 기준으로 사용된다.

*   프론트엔드 RFP 작성
*   백엔드 API 정의
*   개발 범위 및 일정 산정
