# Cherry 기능 정의 문서 (MVP v1.1 기준)

## 1. 기능 정의 문서의 목적

본 문서는 **MVP 정의 문서에서 설정한 목표와 가설을 실제 제품 기능으로 번역**하기 위한 문서이다.

이 문서의 역할은 다음과 같다.

- MVP에서 **반드시 제공되어야 하는 사용자 행동을 명확히 정의**
- 이후 작성될 문서의 기준점 역할
    - 화면 정의서(프론트엔드 RFP)
    - API 정의서(백엔드 RFP)

본 문서에는 **구현 방식, 화면 구성, 기술 스택은 포함하지 않는다.**

---

## 2. MVP에서 검증하고자 하는 핵심 행동

MVP에서 반드시 관찰되어야 할 사용자 행동은 아래 3가지다.

1. 사용자가 **아이돌 굿즈를 탐색**한다
2. 사용자가 **특정 상품에 관심을 표현하거나 판매자와 소통**한다
3. 판매자와 구매자가 **거래를 완료 상태로 전환**한다

이 행동을 가능하게 하는 기능만을 MVP 범위로 정의한다.

---

## 3. 사용자 유형 정의

### 3.1 비로그인 사용자

- 상품을 탐색하고 상세 정보를 확인할 수 있다
- 거래 및 상호작용은 제한된다

### 3.2 로그인 사용자

- 상품을 등록하고 관리할 수 있다
- 다른 사용자와 소통할 수 있다
- 거래 상태를 변경할 수 있다
- 관심 표현 및 후기 작성을 할 수 있다

### 3.3 운영자

- 거래 안전과 서비스 유지에 필요한 최소 개입만 가능하다

---

## 4. 기능 정의 (P0 – MVP 필수)

### 4.1 인증 및 사용자 식별

- 사용자는 계정을 생성하고 로그인할 수 있다
- 사용자는 서비스 내에서 식별 가능한 닉네임을 가진다
- 로그인 여부에 따라 가능한 행동이 구분된다

**검증 목적**

- 실사용자 기반 행동 데이터 확보

---

### 4.2 상품 탐색

- 사용자는 등록된 아이돌 굿즈 상품을 탐색할 수 있다
- 사용자는 조건을 통해 원하는 상품을 좁혀볼 수 있다
- 사용자는 개별 상품의 상세 정보를 확인할 수 있다

**검증 목적**

- 굿즈 특화 탐색 구조가 실제로 사용되는지 확인

---

### 4.3 상품 등록 및 관리

- 사용자는 아이돌 굿즈 상품을 판매 목적으로 등록할 수 있다
- 사용자는 본인이 등록한 상품을 수정하거나 삭제할 수 있다
- 상품은 “판매중” 상태로 시작한다

**검증 목적**

- 판매자가 실제로 상품을 등록할 의지가 있는지 확인

---

### 4.4 상품 상태 관리 (거래 흐름의 핵심)

- 상품은 다음 상태를 가진다
    
    **판매중 → 예약중 → 판매완료**
    
- 판매자는 상품 상태를 변경할 수 있다
- 판매완료 상태의 상품은 거래 종료로 간주한다

**검증 목적**

- 거래가 실제로 ‘완결’되는지 확인

---

### 4.5 사용자 간 커뮤니케이션

- 사용자는 특정 상품을 기준으로 판매자에게 메시지를 보낼 수 있다
- 메시지는 1:1 구조로 전달된다
- 메시지를 통해 거래 협의가 이루어진다

**검증 목적**

- 탐색 → 소통으로 이어지는 흐름이 성립하는지 확인

---

### 4.6 관심 표현(찜)

- 사용자는 상품에 관심을 표시할 수 있다
- 관심 상태는 사용자가 다시 확인할 수 있다

**검증 목적**

- 반복 방문 및 커뮤니티 확장 가능성의 간접 신호 확보

---

### 4.7 후기(선택적이지만 권장)

- 거래 완료 후 사용자는 상대방에 대한 평가를 남길 수 있다
- 평가는 거래 신뢰의 참고 지표로 사용된다

**검증 목적**

- 거래 종료 이후 행동 발생 여부 확인

---

## 5. 운영자 기능 (최소)

- 운영자는 문제 발생 시 상품 또는 사용자를 제한할 수 있다
- 운영자 기능은 서비스 안전을 위한 최소 수준으로 제한한다

---

## 6. MVP 범위에서 제외된 기능 (재확인)

다음 기능은 본 기능 정의 문서 범위에 포함하지 않는다.

- 결제 및 정산
- 배송 및 송장 관리
- 커뮤니티 게시판, 피드
- 추천 알고리즘
- 고급 관리자 기능

---
