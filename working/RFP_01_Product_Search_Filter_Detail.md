# RFP-01 상품 검색 · 필터 · 상세

## 1. 프로젝트 개요

### 목적
Cherry(체리) 아이돌 굿즈 중고 거래 플랫폼 MVP에서 **사용자가 아이돌 굿즈 상품을 탐색하고, 조건을 좁혀보고, 상세 정보를 확인**할 수 있는 모바일 웹뷰 기반 프론트엔드 기능을 구현한다.

본 RFP는 거래 성립 이전 단계인 **“상품 탐색 경험” 검증**을 목표로 한다.

### 대상 사용자
*   비로그인 사용자(게스트)
*   로그인 사용자
    *   ※ 본 RFP 범위 내에서는 두 사용자 간 기능 차이는 없다

## 2. 범위(Scope)

### 2.1 포함 범위 (필수)
1.  상품 목록 화면
2.  상품 검색 기능
3.  상품 필터 기능
4.  상품 정렬 기능
5.  상품 상세 화면
6.  무한 스크롤 기반 리스트 로딩
7.  공통 UI 컴포넌트
    *   상품 카드
    *   필터 UI
    *   Empty / Loading / Error 상태 UI

### 2.2 제외 범위 (명확히 제외)
다음 항목은 이번 RFP 범위에 포함되지 않는다.

*   상품 등록 / 수정 / 삭제
*   찜(관심) 기능
*   채팅 / 거래 상태 변경
*   후기 / 신고
*   장바구니 / 결제 / 에스크로
*   커뮤니티 기능
*   관리자 기능
*   데스크톱 UI 및 반응형 대응

## 3. 플랫폼 및 구현 기준
*   **모바일 웹뷰 전용**
*   단일 화면 기준 (데스크톱/태블릿 고려 없음)
*   MVP 기준으로 **UX 일관성과 구현 단순성 우선**

## 4. 화면 요구사항

### 4.1 상품 목록 화면 (검색 / 필터 포함)
**목적**
*   사용자가 원하는 아이돌 굿즈를 빠르게 탐색할 수 있도록 한다.

**UI 구성 요소**
*   상단 검색 입력 영역
*   필터 버튼 → 필터 패널(바텀시트 또는 풀스크린)
*   정렬 옵션 UI
*   상품 리스트(카드형)
*   무한 스크롤 로딩 인디케이터

**상품 카드 표시 정보 (최소)**
*   대표 이미지(썸네일)
*   상품 제목
*   가격
*   상태 배지
    *   판매중 / 예약중 / 판매완료
*   등록 시점(상대 시간 또는 날짜)

**동작 정의**
*   검색어 입력 후 실행 시 리스트 갱신
*   필터/정렬 변경 시 리스트 초기화 후 재조회
*   스크롤 하단 도달 시 다음 데이터 자동 로딩
*   결과 없음(empty state) UI 제공
*   로딩 중 / 에러 발생 시 명확한 상태 UI 제공

### 4.2 상품 상세 화면
**목적**
*   사용자가 거래 여부를 판단할 수 있을 만큼 충분한 정보를 제공한다.

**UI 구성 요소**
*   이미지 갤러리(다중 이미지 스와이프)
*   상품 제목
*   가격
*   상태
*   카테고리 (데이터가 있을 경우)
*   아티스트 / 그룹 (데이터가 있을 경우)
*   태그(칩 형태)
*   상품 설명
*   판매자 요약 정보
    *   닉네임만 최소 표시

**동작 정의**
*   상품 목록 → 상세 화면 이동
*   상세 화면은 **공유 가능한 고유 URL**을 가진다
*   뒤로가기 시 이전 목록 상태 유지(권장)

## 5. 기능 요구사항

### 5.1 검색 기능 (필수)
**검색 대상 필드 (우선순위)**
1.  상품 제목(title)
2.  태그(tags)
3.  아티스트 / 그룹 (데이터 존재 시)

**검색 UX**
*   키워드 입력 후 엔터 또는 검색 버튼 실행
*   검색어 제거 시 전체 목록 노출

### 5.2 필터 기능 (필수)
*   **판매 상태**: 판매중 / 예약중 / 판매완료 / 전체
*   **카테고리**: 포토카드 / 앨범 / MD 등
*   **가격 범위**: 최소/최대 금액 입력 또는 프리셋
*   **거래 방식**: 택배 / 직거래 (데이터 존재 시)

### 5.3 정렬 기능 (필수)
*   최신순 (기본값)
*   가격 낮은순
*   가격 높은순

### 5.4 무한 스크롤 (필수)
*   초기 로딩 후 스크롤 하단 도달 시 다음 데이터 로딩
*   중복 요청 방지
*   로딩 상태 명확히 표시

## 6. 데이터/API 연동 요구사항
백엔드 API는 이미 존재한다고 가정하며, 프론트엔드는 아래 인터페이스 기준으로 연동한다.

### 6.1 상품 목록 조회
`GET /products`

**Query Parameters (예시)**
*   `q`
*   `status`
*   `categoryId`
*   `artistId` (선택)
*   `tradeType` (선택)
*   `minPrice`
*   `maxPrice`
*   `sort`
*   `cursor` / `limit`

**Response (예시)**
```json
{
  "items": [
    {
      "id": 1,
      "title": "아이브 원영 공방 포카",
      "price": 45000,
      "status": "SELLING",
      "thumbnailUrl": "...",
      "createdAt": "..."
    }
  ],
  "nextCursor": "..."
}
```

### 6.2 상품 상세 조회
`GET /products/{productId}`

**Response (예시)**
```json
{
  "id": 1,
  "title": "...",
  "price": 45000,
  "status": "SELLING",
  "imageUrls": [],
  "description": "...",
  "tags": [],
  "category": "...",
  "artist": "...",
  "seller": {
    "nickname": "..."
  },
  "createdAt": "..."
}
```

## 7. 비기능 요구사항
**성능**
*   썸네일 이미지 lazy loading
*   이미지 로딩 실패 시 대체 UI

**접근성**
*   기본적인 버튼 포커스 및 aria-label 권장

**지원 환경**
*   모바일 최신 Chrome / Safari 기준

## 8. 산출물(Deliverables)
*   상품 목록 화면
*   상품 상세 화면
*   공통 컴포넌트
    *   상품 카드
    *   필터 UI
    *   Empty / Loading / Error UI
*   API 연동 코드
*   간단한 화면별 동작 설명 문서(1~2p)

## 9. 검수 기준(Acceptance Criteria)
*   검색 결과가 조건에 맞게 정확히 반영된다
*   필터/정렬 조합이 정상 동작한다
*   무한 스크롤이 끊김 없이 동작한다
*   결과 없음/에러/로딩 상태가 모두 UI로 표현된다
*   목록 → 상세 → 뒤로가기 시 UX가 자연스럽다
