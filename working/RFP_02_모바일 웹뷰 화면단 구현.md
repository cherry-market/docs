## **Cherry 모바일 웹뷰 화면단 구현 – UI/UX 기준 명시**

---

## **RFP-02 목적**

본 RFP-02의 목적은 **백엔드 연동 없이도** Cherry MVP가

**실제 서비스처럼 보이고 설명 가능한 모바일 웹뷰 화면단을 완성**하는 것이다.

- 상품 탐색 → 주목 상품 인지 → 상세 진입 → 행동(채팅/찜/등록)까지
    
    **화면 흐름이 끊기지 않도록 UI/UX 구조를 확정**한다.
    
- API, 인증, 데이터 저장은 포함하지 않는다.
- 모든 데이터는 **더미(mock) 기준**으로 구현한다.

---

## **0. UI/UX 레퍼런스 명시 (중요)**

본 RFP-02의 모든 화면 UI/UX는

**당근마켓 모바일 앱의 UI/UX를 1차 레퍼런스로 삼아 최대한 유사하게 구현**한다.

- 레이아웃
- 카드 구조
- 여백
- 터치 영역
- 스크롤 방식
- 화면 전환 흐름

은 **당근마켓을 기준으로 한다**.

단, **컬러 포인트 및 브랜드 아이덴티티는 Cherry(체리픽) 색상**을 사용한다.

---

## **1. 채팅 화면 (당근 UI 그대로 사용)**

### **1.1 채팅 목록 화면**

### **UI/UX 기준**

- **당근 채팅 목록 화면과 동일한 구조**
    - 프로필 이미지
    - 닉네임
    - 마지막 메시지
    - 시간 표시
- 상단 필터(All / Selling / Buying / Unread)는 **당근과 동일하게 구성**

### **Cherry 적용**

- 상단 강조 색상
- 활성 필터 색상
- 선택된 채팅 강조 배경
    
    → **체리픽 컬러 사용**
    

---

### **1.2 채팅 상세 화면**

### **UI/UX 기준**

- **당근 채팅 상세 화면과 동일**
    - 좌/우 말풍선
    - 시간 표시
    - 하단 입력창 고정
- 거래 정보 영역(상단 상품 요약) 포함

### **정책**

- ❌ 이모티콘 구현하지 않음
- ❌ 이미지 전송 구현하지 않음
- 텍스트 메시지만 지원

---

## **2. 홈 / 상품 리스트 화면**

---

### **2.1 지금 뜨는 체리픽 (Trending)**

### **역할**

- “요즘 사람들이 많이 보고 있는 게시글”을 강조 노출

### **UI**

- **가로 스크롤(수평 스와이프)**
- 기존 카드 UI 유지
- **카드 세로 비율을 지금보다 조금 더 낮춤**
    - 이미지 높이 축소
    - 정보 밀도는 유지

### **노출 수**

- 최대 10개

---

### **2.2 일반 상품 리스트**

### **UI/UX 기준**

- **당근 상품 리스트 UI와 거의 동일**
- 카드가 **화면 가로폭을 꽉 채우는 구조**
- 이미지 좌측 / 정보 우측 정렬

### **카드 구성**

- 썸네일
- 제목
- 가격
- 상태(판매중/예약/완료)
- 위치/시간(더미)

---

## **3. 상품 등록 진입 방식 (중요)**

### **3.1 등록 버튼**

- **우측 하단 Floating Action Button (+)**
- Cherry 컬러 사용
- 항상 화면 위에 떠 있음

### **3.2 버튼 인터랙션**

- 버튼 클릭 시:
    - **물방울을 누르는 듯한 애니메이션**
    - 버튼이 퍼지듯 확장되며 선택지 노출

### **노출 옵션**

1. 글 작성
2. AI로 글 작성

---

## **4. 글 작성 화면 (일반 작성)**

### **4.1 이미지 업로드 정책**

- 모바일:
    - 휴대폰 카메라 접근
    - 갤러리 접근
    - 권한 요청 필요
- PC:
    - 로컬 이미지 파일 업로드

---

### **4.2 입력 항목**

- 상품 제목
- 상품 설명 (자유 작성)
- 거래 방식 선택
    - ☐ 직거래
    - ☐ 배달
    - ☐ 직거래 + 배달

---

## **5. AI로 글 작성 화면 (핵심 차별 포인트)**

### **5.1 이미지 업로드**

- 일반 글 작성과 동일한 방식

---

### **5.2 필수 입력 항목**

- 상품 기본 설명 (필수)
- 추가 설명 (선택)
    - **최대 200자 이내**

---

### **5.3 AI 성격 선택**

### **선택 방식**

- 키워드 선택 또는 캐릭터 선택(확장 가능)

### **제공 성격(초기)**

1. 친근함
2. 귀여움
3. 깔끔함

---

### **5.4 말투 정책 (중요)**

AI 결과물은 아래 말투 중 하나를 따른다.

- **음슴체**
- **깔끔한 존댓말**
- **여성스러운 말투**
    - 예:
        - “안녕하세용~”
        - “도와드릴게용!”

※ 실제 AI 생성은 RFP-03 범위

※ RFP-02에서는 **결과 예시 더미 텍스트로 UI만 구현**

---

## **6. 정책 요약 (RFP-02 고정)**

- UI/UX는 **당근을 최대한 그대로 따른다**
- Cherry는 색상과 일부 인터랙션으로만 차별
- 기능 구현 ❌
- 화면 흐름과 UX 완성 ⭕
- AI 기능은 **화면 + 상태만 구현**

---

## **7. RFP-02 완료 기준 (보강)**

- 채팅 화면이 당근과 동일하다고 느껴진다
- 체리픽과 일반 상품이 명확히 구분된다
- 상품 등록 버튼의 인터랙션이 직관적이다
- AI 글 작성이 “실제 서비스 기능처럼” 이해된다
